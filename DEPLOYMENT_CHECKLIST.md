## Phase 3: Render Deployment üéØ

### Backend Service:
- [ ] Login to Render (https://dashboard.render.com)
- [ ] Click "New +" ‚Üí "Blueprint"
- [ ] Connect GitHub repository
- [ ] Configure Backend Environment Variables:
  - [ ] `MONGO_URL` - Your MongoDB Atlas connection string
  - [ ] `EMERGENT_LLM_KEY` - `sk-emergent-9E919704404924c86F`
  - [ ] `ADMIN_EMAILS` - Your email address
  - [ ] `JWT_SECRET_KEY` - Auto-generated
- [ ] Note backend URL: `https://noteshub-backend.onrender.com`

### Frontend Service:
- [ ] Deployed automatically via blueprint
- [ ] Configure Frontend Environment Variables:
  - [ ] `VITE_REACT_APP_BACKEND_URL` - Your backend URL
  - [ ] `VITE_API_BASE_URL` - Your backend URL
- [ ] Note frontend URL: `https://noteshub-frontend.onrender.com`

### Update CORS:
- [ ] Go back to backend service
- [ ] Update `CORS_ORIGINS` environment variable
- [ ] Add your frontend URL: `https://noteshub-frontend.onrender.com`
- [ ] Save (will trigger redeploy)

## ‚ö†Ô∏è Common Issues

### "Application failed to respond"
- Backend might be starting up (takes 30-60s on free tier)
- Check logs in Render dashboard

### CORS Errors
- Verify CORS_ORIGINS includes your frontend URL
- Ensure no typos in URLs

### Database Connection Failed
- Check MongoDB Atlas Network Access allows 0.0.0.0/0
- Verify connection string format
- Ensure password doesn't have special characters (or properly encode them)
